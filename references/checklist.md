# Low-Context Checklist

## Session Start
- Read `conductor/tracks.md`
- Read today's `memory/logs/YYYY-MM-DD.md`
- Read only one active track that you will execute next

## During Work
- Update files at milestones only
- Keep each log entry to 3-4 lines
- Record only durable facts: decisions, status, next action, blockers

## Session End
- Add one compact checkpoint in today's log
- Update `conductor/tracks.md` for new status
- Keep chat summary brief and point to files
